## Resolved decisions (critic overrides readers)

- **D1 (CONTRADICTION-1):** All 4 temporal tables (`capability_profile`, `skill_profile`, `capability_skill`, `engineer_skill`) declare their `WITHOUT OVERLAPS` PK `DEFERRABLE INITIALLY IMMEDIATE` **inline**. The CTE delete-then-insert upsert requires it (precedent: `server/priv/migrations/20260624000000_deferrable_upsert_constraints.sql`, inline at `20260628120000:35-40`, `20260629090000:14`). Ignore temporal-command §8's "copy 001_schema.sql verbatim".
- **D2 (VERIFIED-WRONG-1):** `weight` is **Int** (`json.int`/`decode.int`), CHECK 1–3. Only the read-side rollup `proficiency` is Float (`::numeric` → `pog.numeric_decoder()`). Do NOT use `lenient_float_decoder()` for weight.
- **D3 (CONTRADICTION-3):** THREE aggregates per design table (design doc :173-177): `CapabilityCommand` (CreateCapability, DefineCapability, RetireCapability, SetCapabilitySkill, RemoveCapabilitySkill), `SkillCommand` (CreateSkill, DefineSkill, RetireSkill), `EngineerSkillCommand` (AssessSkill). +3 wrapper variants in the envelope.
- **D4 (GAP-3):** Add two retire facts: `CapabilityRetired(capability_id, from)`, `SkillRetired(skill_id, from)` (shape mirrors `UserRoleRevoked`, `fact.gleam:224`). Cascade semantics: **RetireCapability closes capability_profile + that capability's capability_skill rows; RetireSkill closes skill_profile + that skill's capability_skill rows. engineer_skill rows stay open** — matrix/rollup reads join through the skill catalog as-of, so retired skills drop out of reads automatically; closing capability_skill keeps the rollup from counting them. `engineer_skill` IS force-closed on termination (PERIOD FK) — add to `record_departure`.
- **D5 (CONTRADICTION-2):** Client writes split by surface. People-detail **Assess modal = ui.OpKind/OpForm** machinery (detail page already hosts it). **Taxonomy admin page = page-local direct-submit** (access.gleam pattern: page Msg → `api.submit_operation` → refetch) for inline weight cells AND its create/composition modals (steppers don't fit OpForm). `ui.gleam:396-397` "frozen" means pages don't widen the union; Command-backed writes DO add variants — only `OpAssessSkill` is added.
- **D6 (GAP-4):** Phase 1 seeds taxonomy + assessments over the **existing 3-engineer cast**. The ~11-engineer bench (design :243-252) is Phase 2/3; growing it now breaks exact-seed assertions in `server/test/sql_test.gleam:55-186` and `api_test.gleam:135-198`.
- **D7 (GAP-5):** `.github/workflows/test.yml` is stale (references nonexistent migrations/oracle). Gates are `bin/test` + `bin/e2e` only; do not touch CI.
- **D8 (GAP-2):** Codec round-trips go in `server/test/codec_test.gleam` (`shared/test/` is ungated by `bin/test:18-20`). Run `gleam format` in `shared/` manually — also ungated.

```sql
CREATE TABLE capability (id int GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY);

```
2. Profiles (mirror `client_profile` `001_schema.sql:180-185` + inline audit_id per `20260626130000_temporal_rbac.sql:12-45`; D1 deferrable):
```sql
CREATE TABLE capability_profile (
  capability_id int NOT NULL REFERENCES capability(id),
  name          text NOT NULL,
  summary       text NOT NULL,
  defined_during daterange NOT NULL,
  audit_id      bigint REFERENCES event_log(id),
  CONSTRAINT capability_profile_no_overlap
    PRIMARY KEY (capability_id, defined_during WITHOUT OVERLAPS)
    DEFERRABLE INITIALLY IMMEDIATE
);
```
   `skill_profile` identical shape (`skill_id`, `skill_profile_no_overlap`).
3. Weighted M2M (mirror `role_permission` `20260626130000:23-30`, multi-key PK precedent `allocation` `001_schema.sql:141`):
```sql
CREATE TABLE capability_skill (
  capability_id int NOT NULL REFERENCES capability(id),
  skill_id      int NOT NULL REFERENCES skill(id),
  weight        int NOT NULL CONSTRAINT capability_skill_weight_check CHECK (weight BETWEEN 1 AND 3),
  mapped_during daterange NOT NULL,
  audit_id      bigint REFERENCES event_log(id),
  CONSTRAINT capability_skill_no_overlap
    PRIMARY KEY (capability_id, skill_id, mapped_during WITHOUT OVERLAPS)
    DEFERRABLE INITIALLY IMMEDIATE
);
```
   Design: **no PERIOD-FK containment** for taxonomy tables (plain FKs to anchors).
4. Assessments (mirror `engineer_role` `001_schema.sql:50-58`; PERIOD FK IS required here):
```sql
CREATE TABLE engineer_skill (
  engineer_id int NOT NULL,
  skill_id    int NOT NULL REFERENCES skill(id),
  level       int NOT NULL CONSTRAINT engineer_skill_level_check CHECK (level BETWEEN 0 AND 4),
  assessed_during daterange NOT NULL,
  audit_id    bigint REFERENCES event_log(id),
  CONSTRAINT engineer_skill_no_overlap
    PRIMARY KEY (engineer_id, skill_id, assessed_during WITHOUT OVERLAPS)
    DEFERRABLE INITIALLY IMMEDIATE,
  CONSTRAINT engineer_skill_within_employment
    FOREIGN KEY (engineer_id, PERIOD assessed_during)
    REFERENCES employment (engineer_id, PERIOD employed_during)
);
```
5. Audit indexes (mirror `20260623080217_performance_indexes.sql:63`): one `CREATE INDEX <table>_audit_id_idx ON <table> (audit_id);` per new temporal table.
6. Current views (mirror `20260623081652_proration_kernels_and_current_views.sql:61-71`): `capability_current` / `skill_current` = `SELECT <anchor>_id AS id, name, summary FROM <x>_profile WHERE upper_inf(defined_during);`. **No** `engineer_skill_current` (design :144).

Constraint name suffixes are load-bearing: `*_no_overlap`/`*_within_*`/`*_check` → typed errors via `operation.classify_constraint` (`server/src/tempo/server/operation.gleam:90-106`).

Run `bin/migrate`.

- `capability_skill_upsert.sql` — mirror `role/sql/user_role_grant.sql` (keyed open-ended set; DEFERRABLE PK covers re-weight over an open span):
```sql
WITH capped AS (
  DELETE FROM capability_skill
     FOR PORTION OF mapped_during FROM $3::date TO NULL
   WHERE capability_id = $1 AND skill_id = $2
)
INSERT INTO capability_skill (capability_id, skill_id, weight, mapped_during, audit_id)
VALUES ($1, $2, $4, daterange($3::date, NULL, '[)'), $5);
```
- `capability_skill_remove.sql` — keyed cap on (capability_id, skill_id), no audit_id.
- `capability_skill_close_for_capability.sql` — cap ALL rows of one capability (retire cascade): `... FROM $2::date TO NULL WHERE capability_id = $1;`
- Taxonomy snapshot reads (as-of `@> $1::date`, mirror `access/sql/*.sql`): `capability_catalog.sql` (from capability_profile as-of, ordered by name), `skill_catalog.sql`, `capability_skill_matrix.sql` (capability_id, skill_id, weight WHERE `mapped_during @> $1::date`).

**NEW `server/src/tempo/server/skill/sql/`:** `skill_next_id.sql`, `skill_create.sql`, `skill_profile_upsert.sql`, `skill_profile_close.sql`, `capability_skill_close_for_skill.sql` (cap by skill_id — retire cascade).

**NEW `server/src/tempo/server/engineer_skill/sql/`:**
- `engineer_skill_upsert.sql` — mirror `engineer/sql/engineer_role_upsert.sql` verbatim + extra key (upper bound = employment's upper, satisfying the PERIOD FK):
```sql
WITH deleted AS (
  DELETE FROM engineer_skill
     FOR PORTION OF assessed_during FROM $3::date TO NULL
   WHERE engineer_id = $1 AND skill_id = $2
)
INSERT INTO engineer_skill (engineer_id, skill_id, level, assessed_during, audit_id)
SELECT $1, $2, $4, daterange($3::date, upper(employed_during), '[)'), $5
FROM employment
WHERE engineer_id = $1
  AND employed_during @> $3::date;
```
- `engineer_skill_close_all.sql` — mirror `engineer_role_close_all.sql` (termination cascade).
- `skill_matrix.sql` — per-engineer as-of: skill catalog LEFT JOIN engineer_skill ON `assessed_during @> $2::date`, `coalesce(engineer_skill.level, 0) AS level`. Join through `skill_profile ... defined_during @> $2::date` so retired skills drop out (D4).
- `capability_rollup.sql` — weighted average per capability as-of: join capability_profile/capability_skill/engineer_skill all `@> $2::date`, `(sum(coalesce(es.level,0) * cs.weight)::numeric / sum(cs.weight)::numeric) AS proficiency` → squirrel emits `pog.numeric_decoder()` → Float (precedent `pnl/sql/pnl_rows.sql:179-191`, `leave/sql/leave_balance.sql:7-9`).
- `recent_assessments.sql` — history read; **NEVER select raw range bounds** — the trio (`engineer/sql/engineer_role_history.sql:10-18`): `lower(assessed_during) AS valid_from, coalesce(upper(...), lower(...)) AS valid_to, upper_inf(...) AS ongoing`.
